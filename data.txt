# Unclaimed Libraries

[![Python Version](https://img.shields.io/badge/python-≥3.13-blue.svg)](https://python.org)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)
[![CLI](https://img.shields.io/badge/CLI-unclib-orange)]()

**Unclaimed Libraries** is a CLI tool that detects **unused dependencies** in your Python project.
It scans your `pyproject.toml`, analyzes all imports in your source code, filters out standard library and local modules, and even checks PyPI metadata to skip development‑only packages (linters, testing frameworks, CLI tools, etc.).

---

## ✨ Features

- ✅ **Accurate detection** – understands `import`, `from ... import`, dynamic imports (`importlib.import_module`, `__import__`).
- 🧠 **Smart filtering** – automatically ignores:
  - Python standard library modules
  - local modules/packages (including namespace packages)
  - development‑oriented packages (linters, test frameworks, static analyzers, CLI helpers) – using **PyPI classifiers**
- ⚡ **Fast** – parallel processing of files (configurable thread pool).
- 📦 **Supports modern project metadata** – PEP 621 (`pyproject.toml`) and Poetry.
- 🎨 **Nice terminal output** – coloured tables with `rich`.

🚀 Usage

The CLI command is unclib.
bash
unclib [PROJECT_PATH] [--exclude DIR1 DIR2 ...]
Argument	Description
PROJECT_PATH	Path to your project root (default: .)
--exclude	One or more directory names to skip (e.g. tests docs)
Examples

bash
# Analyze current directory
unclib

# Analyze a specific project
unclib /path/to/myproject

# Exclude additional directories
unclib . --exclude legacy scripts
Exit codes

Code	Meaning
0	No unused dependencies found ✅
1	Unused dependencies found ❌
2	Error during analysis (misconfiguration, missing files, etc.)

⚙️ How it works

Reads pyproject.toml – extracts all declared dependencies (PEP 621 project.dependencies + Poetry sections).
Walks through every .py file – parses the AST to collect all imported top‑level names.
Filters out:
__future__
Python standard library modules (using sys.stdlib_module_names plus a curated list)
Local modules/packages (detected by building a module map of your own code)
Classifies the remaining candidates – queries PyPI JSON API to read classifiers:
library – only these are reported as unused.
linter, testing, static_analyzer, cli_tool, framework, types – considered “assistants” and skipped.
Prints the final list of truly unused libraries.
Why skip assistants?
Packages like pytest, black, mypy, ruff, click are often installed but never directly imported in the code – they are used via CLI or tool configuration. Unclaimed Libraries ignores them so you don’t get false positives.

Невостребованные библиотеки

Unclaimed Libraries – это CLI‑инструмент для поиска неиспользуемых зависимостей в Python‑проектах.
Он анализирует pyproject.toml, просматривает все импорты в коде, отфильтровывает стандартную библиотеку и локальные модули, а также обращается к PyPI, чтобы пропустить пакеты, предназначенные только для разработки (линтеры, тестовые фреймворки, CLI‑утилиты и т.д.).

✨ Возможности

✅ Точное обнаружение – обрабатывает import, from ... import, динамические импорты (importlib.import_module, __import__).
🧠 Умная фильтрация – автоматически игнорирует:
модули стандартной библиотеки Python
локальные модули/пакеты (включая namespace packages)
пакеты для разработки (линтеры, тесты, статические анализаторы, CLI‑помощники) – используя классификаторы PyPI.
⚡ Быстро – параллельная обработка файлов (настраиваемый пул потоков).

⚙️ Как это работает

Читает pyproject.toml – извлекает все объявленные зависимости (PEP 621 project.dependencies + секции Poetry).
Обходит все .py‑файлы – парсит AST и собирает все импортированные имена верхнего уровня.
Отфильтровывает:
__future__
модули стандартной библиотеки Python (используется sys.stdlib_module_names + дополненный список)
локальные модули/пакеты (строится карта всех ваших модулей)
Классифицирует оставшиеся кандидаты – запрашивает PyPI JSON API, чтобы прочитать классификаторы:
library – только эти выводятся как неиспользуемые.
linter, testing, static_analyzer, cli_tool, framework, types – считаются «помощниками» и пропускаются.
Выводит итоговый список действительно неиспользуемых библиотек.
Почему пропускаются «помощники»?
Пакеты вроде pytest, black, mypy, ruff, click часто устанавливаются, но никогда не импортируются напрямую – они используются через CLI или конфигурацию инструментов. Unclaimed Libraries игнорирует их, чтобы избежать ложных срабатываний.
